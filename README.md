# Home Assistant — Automation Ideas & Configuration Patterns

Этот репозиторий содержит **идеи, архитектурные решения и шаблоны конфигураций** для Home Assistant.

⚠️ **Это не полный бэкап системы.**  
Полный бэкап (включая secrets, базы данных и runtime-файлы) хранится отдельно.

---

## 🎯 Цель репозитория

- Хранить **чистую логику автоматизаций**
- Фиксировать **архитектурные решения и идеи**
- Вести **историю изменений конфигурации**
- Использовать Git как инструмент проектирования, а не восстановления системы

---

## 📁 Что находится в репозитории

В репозиторий намеренно попадают **только YAML-файлы из корня `/config`**, например:

- `configuration.yaml`
- `automations.yaml`
- `scripts.yaml`
- `scenes.yaml`
- другие root-конфигурации

### Что здесь **отсутствует намеренно**
- `secrets.yaml`
- runtime-файлы (`.db`, `.log`, cache)
- интеграции и кастомные компоненты
- файлы видеонаблюдения и потоков
- конфигурации, привязанные к конкретному оборудованию

---

## 🔐 Секреты и персональные данные

Все чувствительные данные (пароли, токены, URL камер, chat_id и т.п.):

- вынесены в `secrets.yaml`
- **не хранятся в этом репозитории**
- не предназначены для публикации

---

## 🧠 Основные идеи, используемые в конфигурации

- Разделение:
  - **Git → идеи и логика**
  - **Backup → восстановление системы**
- Использование `template`, `choose`, `variables`
- Явная синхронизация состояний (watchdog / timer-based sync)
- Минимизация жёстких связей с конкретным оборудованием
- Явное управление побочными эффектами (уведомления, перезапуски)

---

## 🔄 Как использовать этот репозиторий

Этот репозиторий **не предназначен для прямого деплоя**.

Рекомендуемый сценарий:
1. Использовать конфигурации как reference
2. Адаптировать под свою систему
3. Добавлять собственные secrets локально
4. Проверять через Home Assistant UI

---

Структура и подходы могут эволюционировать по мере накопления опыта.
